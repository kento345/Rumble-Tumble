# Unity レビュー基準

Unity のランタイム挙動・アセット・シーン・プレハブ・MonoBehaviour・フレームパフォーマンスに触れるレビューで参照する。

---

## ライフサイクルと初期化

**確認ポイント**
- `Awake` / `OnEnable` / `Start` / `Update` / `FixedUpdate` / `LateUpdate` / `OnDisable` / `OnDestroy` が明確な所有権で使われているか
- 異なるシーン・プレハブ間のオブジェクトで初期化順の前提が成立しているか
- イベントの購読と解除が対応するライフサイクルメソッドで行われているか (`OnEnable` ↔ `OnDisable`)
- シーンリロード・リスポーン・リプレイなど一部の開始パスでしかリセットされない状態がないか

---

## シリアライズされた参照とアセット

**確認ポイント**
- `SerializeField` の参照はシーン/プレハブの配線が欠けていると実行時に null になる
- Inspector 依存の参照には、失敗が診断しにくい場合にバリデーション・安全なフォールバック・明示的なエラー出力を用意する
- フィールドのリネーム・列挙値の変更・インデックス変更はシリアライズ済みのアセット挙動をサイレントに変える可能性がある

---

## シーンとプレハブの統合

**確認ポイント**
- スクリプトが特定のシーン階層・オブジェクト名・タグ・レイヤー・コンポーネント配置を前提にしていないか
- `Find` / `FindObjectOfType` / タグ検索 / 名前検索は、依存が意図的にグローバルかつ安定でない限り脆弱とみなす
- プレハブバリアントは古い値を継承する可能性がある。問題の確認がプレハブ/シーンアセット側でも必要か明示する

---

## フレームパフォーマンス

**ホットパスで避けるべき処理**
- `GetComponent` の毎フレーム呼び出し — GC と検索コスト
- `Find` / `FindObjectOfType` — 高コストのシーン走査
- LINQ・文字列操作 — 毎フレームアロケーション
- コレクションの毎フレーム再構築 — GC スパイク

> 最適化を推奨するときは、必ずホットパスまたは想定される規模の問題と結びつける。根拠のない最適化提案はしない。

---

## 物理・入力・時間

**基本ルール**
- 入力の読み取りは `Update`、Rigidbody の操作は `FixedUpdate`
- `Time.deltaTime` / `Time.fixedDeltaTime` / アンスケール時間の混在に注意
- コリジョン/トリガーのロジックで確認すること:
  - レイヤーフィルタリングの欠如
  - 重複イベント
  - オブジェクト無効化・破棄時のクリーンアップ

---

## コルーチン・非同期・状態遷移

**リスクパターン**
- ガードなしで繰り返し `StartCoroutine` — コルーチンの積み上がり
- 文字列指定の `StopCoroutine` またはハンドル紛失 — キャンセルが壊れやすい
- 非同期コールバック内でオブジェクト参照 — 対象が破棄済みの可能性

**要注意のフロー**: シーン遷移・リスポーン・オブジェクトの無効化/有効化サイクル・ゲームオーバー

---

## 静的状態・シングルトン・ScriptableObject

**確認ポイント**
- 静的フィールドとシングルトンインスタンスは想定より長く生存し、クロスシーンの状態リークを起こしやすい
- ScriptableObject は共有ミュータブル状態になりうる。ランタイムの変更が意図的か、正しくリセットされるかを確認する
- ドメインリロードと Enter Play Mode の設定は、静的状態のリセットの信頼性に影響する

---

## Unity の null と破棄のセマンティクス

**確認ポイント**
- 破棄された Unity オブジェクトはマネージド参照を持ちながら null と比較される場合がある
- 破棄・プーリング・シーンアンロードをまたぐ参照の保持は慎重に確認する
- null ガードは「シリアライズ参照の欠如」と「ランタイムでの破棄済みオブジェクト」の両方を対象にする

---

## アニメーション・オーディオ・VFX・イベント

**文字列依存の設定値**
- Animator パラメータ名（例: `animator.SetBool("IsCharge", ...)`）
- アニメーションイベント名
- Audio Mixer グループ名
- VFX オブジェクトパス（Resources パスなど）
- Timeline シグナル名

名前の不一致がデバッグ困難な場合は、定数・ラッパー・バリデーション・エディタチェックの追加を提案する。アニメーションイベントがゲームプレイ状態を変更する場合はコールバック順を確認する。

---

## ビルドとプラットフォーム

**確認ポイント**
- エディタ専用の API は `#if UNITY_EDITOR` で囲むか、ランタイムアセンブリから除外する
- プラットフォーム固有の入力・解像度・ファイルパス・パフォーマンス前提は該当する場合に指摘する
- エディタのプレイモードだけでなく実機ビルドでの確認が必要な指摘はその旨を明示する
